## Development conventions for research projects

- **Consistent Structure**: Follow existing package organization in `selfcheckgpt/`
- **Clear Documentation**: README.md explains concepts and usage
- **Version Control**: Use clear commit messages describing research progress
- **No Secrets in Git**: Keep API keys in environment variables or local config
- **Minimal Dependencies**: Add dependencies only when necessary
- **Demo Notebooks**: Use Jupyter notebooks to demonstrate new features

### Not Applicable
- No code review process (solo research)
- No feature flags needed
- No changelog required
- Pragmatic over formal processes
